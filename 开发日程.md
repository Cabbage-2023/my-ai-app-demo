## 📅 2026-05-15 阶段：突破 Bangumi 会话反爬机制

- **踩坑记录**：Axios 请求频繁遭遇 502/403，发现核心 Session ID `chii_sid` 被服务器强制高频刷新（约 1 分钟过期），导致传统静态 Cookie 爬虫彻底瘫痪。
- **攻坚方案**：
  1. 放弃朴素的 HTTP Client 硬怼，引入 Puppeteer 托管真实 Chromium 浏览器。
  2. 实现了"注入初始 Cookie → 模拟人类浏览行为随机延迟（3-5s）→ 浏览器自动维持和更新 Session"的自动化方案。
  3. 通过配置 `.npmrc` 切换至阿里云国内镜像，解决了 Chromium 150MB 底层二进制包在国内环境下的网络下载卡死问题。

## 📅 2026-05-16 阶段：数据画像与全量清洗 + 分块策略定稿

- **当时现状**：全量抓取了约 50,000 条多维数据（短评、长评、角色等）。长评发现只有 128 字摘要，属于严重残缺数据。
- **神级数字（核心吹点）**：跑了 `analyze-data.ts` 拿到关键画像：
  - 角色评论：中位数 11 字，≤15 字噪音占比高达 **64%**
  - 短评：≤15 字噪音占比 **29%**
  - 长评：78% 大于 500 字，中位数 1288 字，最长超 8 万字
- **硬核决策**：
  1. 绝不留技术债，连夜编写 `patch-reviews.ts` 补全 1263 篇长评全文，打通 Parent-Child 分块链路。
  2. **差异化阈值去噪**：短评去噪阈值设为 15 字（剔除 29% 垃圾）；角色评论果断提至 20 字，定向爆破 64% 低语义情绪水评。
  3. **保持原子独立性**：短评与角色评论坚持"单条成块，绝不拼凑"，规避多用户主观视角强行交织带来的多主体交叉污染。
  4. **长文本标点降级**：对长评全文定制 500 窗口 + 50 重叠度，启用 `\n\n → \n → 。 → ，` 的强语义标点递归切分器。
- **阶段成果**：全站数据从 50,000 条粗糙文本，精炼为约 **29,000 个高质量 Chunk**。帮 Atlas M0 省了约 60% 存储成本，同时净化了向量空间。

## 📅 2026-05-17 ~ 5-18 凌晨：Phase 1 收尾

### 阶段 1：Atlas → 腾讯云自托管 Qdrant 迁移

- **动因**：Atlas M0 免费层写入 IOPS 瓶颈（3 万条数小时），且 MongoDB Community 不支持 `$vectorSearch`。
- **方案**：服务器已有 wenjuan-mongo Docker 容器内新建 `ai_rag` 库 + Qdrant 容器（端口 3933 开放，27017 关闭）。
- **架构简化**：Qdrant payload 直接存 content+metadata，搜索返回即用，无需二次查 MongoDB。MongoDB 仅做备份。
- **结果**：SCP 传输 367MB 数据，Qdrant 写入 28,602 条向量，MongoDB 写入 27,444 条文档（去重 3,757）。

### 阶段 2：评估脚本迭代（30 → 82 → 150 QA 对）

- **V1（30 QA）**：事实类模糊 100%，观点类模糊 85%。发现第一版 QA 纯查表→重构为 fact/opinion/comparison 三类。
- **V2（82 QA）**：新增 Precision@K、TypeAcc、Diversity 指标。事实类 100%，观点类模糊 Hit@3=78.4%，对比类 81.3%。
- **V3（150 QA）**：修复对比类匹配逻辑（要求所有 expect 同时出现），对比类模糊 Hit@3=60.0%（回归真实）。Prec@5=60.3%。

### 阶段 3：NameCN 别名系统（1.11）

- 手工维护别名映射（~20 游戏短名 + ~12 品牌 + ~10 角色中文名），在 `load-to-qdrant.ts` 中用 `computeNameAliases()` 写入 payload 字段 `gameAliases`/`charNameCN`/`producer`。
- 别名后评估：严格 Hit@3=74.0%（+4.7pp），模糊 Hit@3=89.3%（+5.3pp）。素晴日（O16）从 ✗ → ✓ 严格第 1 位。
- **警示**：Qdrant PUT /points/payload 是全量覆盖不是 merge，导致 8939 点丢失字段，后用 PUT /points 全量恢复。

### 阶段 4：LLM 辅助别名 + 补爬（1.12 ~ 1.13）

- `scripts/utils/suggest-aliases.ts` 调 LLM 为游戏中角色推荐常用译名/俗称，输出 review 文件手工确认。
- 发现 rank-page-2/5/7 爬虫缓存为 502 错误页（`fetchHTML` 未检查 `res.ok`），导致 ~70 游戏永久缺失。
- 补爬 51 游戏 + 增量 embedding，Qdrant 升至 34,984 点。Merge 别名：114 游戏别名 key + 770 角色别名 key。
- `computeNameAliases()` 新增 `charAliases` 字段，evaluate.ts 同步支持。

### 阶段 5：全量短评+长评补爬 & JSONL 迁移（1.14，5-18 凌晨）

- **动因**：短评只爬了前 3 页（热门游戏有几十上百页），长评只有第 1 页无分页。补爬后缓存膨胀至 633MB，`JSON.stringify` 触发 `RangeError: Invalid string length`。
- **方案**：`crawl-more-pages.ts` 遍历 194 游戏补爬短评第 4~20 页 + 长评第 1~10 页；缓存格式全面从单文件 JSON 切换为**流式 JSONL**（逐行读写，不限文件大小）。
- **附带优化**：短评去噪阈值从 15 提升至 30，过滤比例 29% → ~45%；`embed-supplement.ts` / `load-to-qdrant.ts` / `load-cache-to-db.ts` 全部 JSONL 化。
- **结果**：Qdrant 从 34,984 → **49,367** 点（+40.6%），新增 23,941 短评 + 1,050 长评。
- **评估**：数据量增长 40%，模糊 Hit@5 维持 95.3% 未降质；对比类模糊 Hit@5 +4pp（84→88%）。详细指标见项目计划 1.14。

### Phase 1 完成总结

- **数据**：193 游戏、34,984 向量、高质量清洗去噪（去噪率 51%）
- **评估**：150 QA、全 Phase 可重跑设计
- **别名**：114 游戏别名 + 770 角色别名，系统级嵌入灌库流程
- **总量**：Phase 0 爬虫 + Phase 1 分块/灌库/别名/评估，完整闭环已跑通

## 📅 2026-05-18

### Phase 1 收尾 — 工程整理 + RAG 评估优化

#### 阶段 1：Scripts 文件夹重构（1.15）

- **动因**：`scripts/` 根目录 16 个 .ts 平铺 + `lib/` 14 个 .ts 平铺，爬虫/评估/ETL/工具混在一起。
- **方案**：按功能域拆分为 `crawl/`、`eval/`、`etl/`、`utils/` 子目录，lib 也对应拆分为 `lib/crawl/`、`lib/qdrant.ts`、`lib/name-aliases.ts`。
- **Import 更新**：约 35 处路径调整，全部用 `replace_all` / Agent 批量处理。
- **文档同步**：项目计划.md 约 50 处路径引用更新。

#### 阶段 2：RAG 生成质量评估 Step 1（1.16）

- **背景**：已有评估只覆盖检索阶段（HitRate/MRR），缺失对生成质量的量化。新增 Faithfulness + Answer Relevance 两个 LLM-as-judge 指标。
- **做法**：`scripts/eval/evaluate-generation.ts`，复用 150 QA pairs → search → 组装 context → DeepSeek 生成回答 → LLM 自写 judge 打分。JSONL 缓存，450 次调用约 20 分钟。
- **结果**：
  - Faithfulness：平均 **98.3%**，中位数 100%。模型严格遵循 prompt，基本不编造。
  - Answer Relevance：平均 **89.5%**，中位数 100%。多数回答质量好，低分根因是数据缺失。
  - 对比类最低（82.8%），因 context 需同时覆盖多实体，符合预期。
- **结论**：生成质量基线已建立。Phase 2 后接入 Context Recall/Precision（RAGAS）与格式校验（DeepEval）。

### Phase 2 — 自查询检索 + LangGraph 集成 + 前端改造

#### Self-Querying Filter 迭代（V1→V3）

**动因**：TypeAcc 0.4%（5 种 type 混杂），对比类 AnsRelevance 82.8%。不是搜不到，是纯度不够。

**迭代路线**：
- **V1 — must filter**：严格 Hit@1 +18pp（58%→76%），但模糊 Hit@5 下降（95.3%→88%）。精确匹配漏别名
- **V2 — 别名展开（min_should）**：严格 Hit@1 76%→83.3%，事实类 100% 严格命中，模糊 Hit@5 恢复（96%）。踩坑：Qdrant min_should 需嵌套 `{ min_should: { conditions, min_count } }` 对象格式，非 Filter 顶层属性
- **V3 — 多路召回 round-robin**：对比类严格 Hit@3/Hit@5 从 44%/76% → 100%/100%。踩坑：score-sort 合并被高分游戏 dominate（C10 苍之彼方 top 5 全是 EXTRA2），round-robin 交错修复

**最终指标**：

| 指标 | Baseline | V3 | Δ |
|------|----------|-----|-----|
| 严格 Hit@1 | 58.0% | **83.3%** | +25.3pp |
| 严格 Hit@3 | 72.7% | **98.7%** | +26.0pp |
| 严格 Hit@5 | 80.7% | **98.7%** | +18.0pp |
| MRR | 0.680 | **0.909** | +0.229 |
| AnsRelevance | 89.5% | **92.9%** | +3.4pp |
| 对比类 AnsRelevance | 82.8% | **92.4%** | +9.6pp |

**踩坑记录**：
1. Qdrant `min_should` 非 Filter 顶层属性，须嵌套为 `must: [{ min_should: { conditions, min_count } }]` 内层 condition
2. 多路召回必须 round-robin 交错而非 score 排序，否则单一高分游戏 dominate 前列
3. 生成评估的 JSONL cache key 只含 question text，测试 filter 需清缓存否则命中旧结果

**改动范围**：`src/lib/qdrant.ts`（searchSimilar 加 filter）→ `src/app/api/chat/route.ts`（getInformation 加 filter schema）→ `scripts/eval/evaluate.ts`（selfQueryingStrategy 三步迭代）→ `scripts/eval/evaluate-generation.ts`（searchAndAssemble 加 filter + 多路召回）

**结论**：Phase 2 收尾。剩余缺口（对比类 Faithfulness 86%→下降、2 题数据缺失）归入 Phase 3 reranker + 补爬范畴。

#### Vercel AI SDK → LangGraph 迁移

**动因**：旧 route.ts 用 Vercel AI SDK `streamText()` + 直接 tool() 定义，检索逻辑与路由耦合。LangGraph 提供 StateGraph 编排、工具节点、检查点持久化、条件路由。

**迁移要点**：
- 放弃 `streamText`，改用 LangGraph `agent.stream()` 按节点粒度输出
- 保留 Vercel AI SDK 前端部分（`useChat` + `@ai-sdk/react`），通过 `createUIMessageStreamResponse` 包装 ReadableStream 保持兼容
- 手动构造 UI message chunk 协议：`text-start → text-delta → text-end → finish`
- 自定义 ReadableStream 监听 event.agent（发射 tool-input-available + text 序列）和 event.tools（发射 tool-output-available）
- 前端 `page.tsx` 新增 `tool-getInformation`、`tool-addResource` 等渲染分支，tool 卡片支持合并展示、计数去重、来源标签

**踩坑**（6 项已解决，4 项待 Phase 3）：
1. ✅ Tool 返回值序列化（DeepSeek 要求 content 为字符串）
2. ✅ DeepSeek Thinking Mode 不兼容（`modelKwargs: { thinking: { type: "disabled" } }`）
3. ✅ SSE 协议格式不匹配（text-start/delta/end 三段式）
4. ✅ 工具卡片碎片化（合并展示 + 去重 + 空结果隐藏）
5. ✅ 知识库卡片计数跳变（等全部完成再渲染 + 上限 5 条 + 来源标签细化）
6. ✅ 流式卡死前端防抖（status 检查 + stop 按钮）
7. ⏳ 角色名别名检索漏召回（Phase 3 别名映射表）
8. ⏳ 非流式输出（节点粒度限制，Phase 3/4）
9. ⏳ 检索命中但不相关/幻觉召回（Phase 3 reranker + 置信度阈值）
10. ⏳ 后端级中止（AbortController 透传 LangGraph）

**文件**：新增 `src/lib/ai/langgraph/`（graph.ts / tools.ts / nodes.ts / memory.ts），改造 `src/app/api/chat/route.ts`、`src/app/page.tsx`
